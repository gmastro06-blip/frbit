# Evolución del Anti-Cheat en Tibia (1997–2026)

## 1. Moderación Humana (GMs)
- 1997–2007: GMs y Counsellors revisaban denuncias manualmente.
- Limitaciones: corrupción, lentitud, subjetividad.

## 2. Herramientas Automáticas (ADT)
- 2007–2017: Automated Detection Tools (ADT) para detectar patrones de botting (movimientos, clicks, horarios).
- Baneos masivos periódicos, pero bots evolucionaban para simular comportamiento humano.

## 3. BattlEye (desde 2017)
- Integración de BattlEye, anti-cheat de nivel kernel.
- Detección de inyección de DLLs, memory reading, firmas de bots conocidos.
- Actualizaciones frecuentes, baneos automáticos y silenciosos.
- Limitaciones: bots pixel/IA, evasión mediante VMs, hardware spoofing.

## 4. Estado Actual (2026)
- BattlEye sigue activo, pero la evasión es una carrera armamentista.
- Uso de machine learning para detectar patrones anómalos.
- Comunidad de bots desarrolla técnicas anti-detección avanzadas (obfuscation, randomización, bots distribuidos).
